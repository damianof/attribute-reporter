# Attribute Reporter v2.1.0 Plan

## Problem
Add three UX improvements in priority order: negate/missing filter, element-type filter, and Playwright export.

---

## Phase 1 — aria-label badge + Negate/Missing filter

Small, self-contained changes to `AttributeReporter.component.vue` only.

1. **p1-aria-badge** — Add `"aria-label"` to `attributeBadges` array
2. **p1-negate-toggle** — Add a "∅ missing" toggle button in the attribute row; style like existing badges
3. **p1-negate-filter** — When toggle is on, filter `sortedReportItems` to `attributeNotSet === true` only (local state, no store change)
4. **p1-negate-badge-count** — item-count-badge shows filtered count; copy-all JSON respects the filter

---

## Phase 2 — Element-type filter

Dynamic badge row derived from current results. Changes to `AttributeReporter.component.vue` and minor CSS in `App.vue`.

1. **p2-element-types** — Compute sorted unique `item.name` list from `sortedReportItems`
2. **p2-element-filter-ui** — Render element-type badges row (only when 2+ distinct types present)
3. **p2-element-filter-multi** — Multi-select toggle; empty selection = show all
4. **p2-element-filter-reset** — Clear active element filter when target attribute name changes

---

## Phase 3 — Playwright locator export

Single button, no new UI components.

1. **p3-playwright-format** — Format each item as `page.getByTestId("val")` (for `data-testid`) or `page.locator('[attr="val"]')` otherwise
2. **p3-playwright-button** — "Copy as Playwright" button near copy-all badge, with same flash mechanic; outputs one locator per line

---

## Notes
- All filter state stays local to `AttributeReporter.component.vue` — no reactive store changes needed for any phase
- Phase 1 and 2 filters should compose (negate + element type can both be active simultaneously)
- Playwright export uses the *currently filtered* list
